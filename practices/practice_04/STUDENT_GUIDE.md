## Краткий обзор тем

### MCP (Model Context Protocol)

Открытый стандарт расширения возможностей LLM. Аналогия: USB для AI — один протокол, множество инструментов. На практике подключим 4 MCP-сервера: PostgreSQL, Context7, Memory Bank, Sequential Thinking.

### Миграция на PostgreSQL

Практическое задание: с помощью Roo Code мигрировать in-memory хранилище подписок на PostgreSQL. Используем режимы architect → code и проверяем результат через MCP.

### Git Worktree

Запуск нескольких агентов параллельно в изолированных рабочих каталогах. Каждый worktree — отдельная ветка, отдельный экземпляр Roo Code, отдельная задача.

### AI Code Review

Демо: Roo Code Cloud — бесплатные Cloud Agents для автоматического ревью PR в GitHub.

Подробности, конфиги и пошаговые демонстрации — на практике.

## Домашнее задание (10 баллов)

### 1. Завершить миграцию на PostgreSQL + Alembic — 4 балла

- Все эндпоинты работают с PostgreSQL
- Настроить Alembic: `alembic init`, первая миграция, `alembic upgrade head` поднимает схему с нуля
- Docker Compose для PostgreSQL
- Используйте `use context7` при работе с Alembic — агент подтянет актуальную документацию

### 2. Написать свой MCP-сервер — 3 балла

Минимальный MCP-сервер на Python (`fastmcp`), предоставляющий 1–2 инструмента.

Примеры идей:
- Поиск TODO/FIXME в коде
- Обёртка над REST API
- Текущая погода

Цель — понять протокол изнутри, а не как чёрный ящик.

### 3. Эксперимент: Sequential Thinking на практике — 1 балла

- Затащите строгий линтер: `ruff check --select ALL`
- Почините ошибки в два захода: **без** Sequential Thinking MCP и **с ним**
- Скриншоты обоих подходов + рефлексия: что изменилось? Быстрее? Качественнее?

### 4. Рефлексия в task.py — 1 балл

- Что нового узнали?
- Какой MCP-сервер самый полезный и почему?
- Где агент ошибался при работе с PostgreSQL?

### 5. Slash-команда для code review - 1 балл

Создать `.roo/commands/review.md` — slash-команду для code review текущего файла по вашему чек-листу (стиль, безопасность, тесты). Приложите скриншот результата.
