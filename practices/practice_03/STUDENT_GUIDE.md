## О чём практика

Вы научитесь использовать AI IDE (LLM, интегрированную в среду разработки) для решения реальных задач разработки. Мы разберём два подхода — интуитивный «быстрый старт» и инженерный с правилами, режимами и multi-chat workflow.

**Инструмент:** Roo Code — open-source расширение для VS Code.

## Краткий обзор тем

### Интуитивный подход (GET /weather)

Один чат, один промпт — знакомство с интерфейсом Roo Code и базовая работа с AI-агентом.

### Инженерный подход (POST /subscribe)

- **Rules** — файл `.roo/rules.md` для борьбы с недетерминированностью LLM
- **Modes** — режимы architect, code, ask и зачем переключаться между ними
- **Multi-chat workflow** — 1 задача = 1 чат, зачем это нужно
- **Auto-approve** — настройка для ускорения работы с агентом

### Ключевые концепции

- Три подхода к разработке: No LLM coding, Deep coding, Vibe coding
- Гигиена контекста (качество падает после 70–80k токенов)
- Правило 90/10 (планирование важнее генерации)
- Детерминированный feedback loop (код → линтер → тесты → авто-фикс)
- Эволюция правил (.roo/AGENTS.md)

Подробности, примеры промптов и демонстрации — на практике.

## Домашнее задание (11 баллов)

### Обязательная часть (10 баллов)

#### 1. Реализовать 2 эндпоинта — 4 балла

- `DELETE /subscribe/{id}`
- `GET /subscriptions`

Главное: **экспериментируйте с LLM** в процессе. Используйте инженерный подход (architect → code), пробуйте разные промпты.

#### 2. Создать ≥3 полезных агента в Roo Code — 3 балла

Агенты должны иметь практический смысл, например:
- Агент для тестирования
- Агент для анализа архитектуры
- Агент для QA notes
- и т.д.

#### 3. Мигрировать правила в agents.md — 1 балл

- Ознакомьтесь с форматом `agents.md` (универсальный формат для AI IDE)
- Перенесите правила из `.roo` в `agents.md`

#### 4. Рефлексия в task.py — 1 балла

Отрефлексируйте процесс: реализацию эндпоинтов и/или вайб-кодинг из доп. задания.

### Дополнительное задание (+2 балла)

**Навайбкодите что угодно на новом/малознакомом ЯП:**
- Лендинг/сайт (рекомендую V0)
- CLI-тулза
- Либа на Rust для Python
- Всё, что угодно

> **Проверка:** запись экрана с демонстрацией процесса.

### Итого: 9 + 2 дополнительных = 11 баллов
