## 📝 Пошаговая инструкция

### Подготовка

1. Соберитесь в группу (3-4 человека)
2. Откройте `task.py` в редакторе — это ваша рабочая тетрадь
3. **Что открыть в `task.py`:**
   - Секция `STUDENT_INFO` — заполните информацию о группе
   - Секция `EVENT_STORMING` — для результатов event storming
   - Секция `CJM` — для Customer Journey Map
   - Секция `ROADMAP` — для roadmap по версиям
   - Секция `EPICS` — для списка Epics
   - Секция `USER_STORIES` — для User Stories
   - Секция `DEFINITION_OF_READY` — для DoR
   - Секция `ARCHITECTURE_COMPONENTS` — для компонентов системы
   - Секция `ADR` — для Architecture Decision Record
   - Секция `MERMAID_CODE` — для кода Mermaid диаграммы
   - Секция `DEFINITION_OF_DONE` — для DoD
   - Секция `TEST_PLAN` — для плана тестирования
   - Секция `FUNCTIONAL_DELIVERY` — для Jira-тикетов
   - Секция `PROMPT_LOGS` — для записи всех промптов
   - Секция `REFLECTION` — для рефлексии
4. Один человек открывает LLM-чат (будет вести диалог)
5. Откройте доску для визуализации (Miro/Excalidraw/Draw.io)
6. Распределите задачи между участниками группы

### Этап 1: CJM и UX через event storming

#### Задание 1.1: Event storming (light)

**Что делать:**

```
1. Откройте LLM-чат
2. Попросите AI провести лёгкий event storming
3. Получите список Actors / Commands / Domain Events
4. Обсудите: не хватает ли ключевого события? нет ли "воды"?
5. Скопируйте финальный вариант в task.py
```

**Пример промпта:**

```
Контекст: WeatherService — REST API сервис уведомлений о погоде (подписка на города, управление подписками, получение текущей погоды из внешнего API).
Задача: Проведи лёгкий event storming: перечисли 5–7 domain events (в прошедшем времени), 3–5 commands и 3–5 actors.
Формат: Markdown с заголовками Actors, Commands, Domain Events.
```

**Куда вставить результат:**

- В `task.py` → переменная `EVENT_STORMING`
- На доску → блок «Event storming»

**⚠️ Типичные ошибки:**

- События сформулированы в будущем времени («создадим подписку») вместо прошедшего («SubscriptionCreated»)
- Слишком много технических деталей вместо бизнес-событий
- Забыли зафиксировать промпт в `PROMPT_LOGS`

#### Задание 1.2: CJM + Roadmap

**Что делать:**

```
1. Попросите AI собрать CJM по основному сценарию "подписка → уведомления"
2. Попросите AI предложить roadmap по версиям v1.0/v1.1/v2.0
3. Обсудите, что будет минимальным слайсом v1.0
4. Скопируйте результат в task.py
```

**Пример промпта:**

```
Собери CJM (6–8 шагов) для сценария "подписка → уведомления" в WeatherService.
Затем предложи roadmap по версиям v1.0/v1.1/v2.0: что войдёт в каждую версию и почему (слайс по ценности).
```

**Куда вставить результат:**

- CJM → `task.py` → переменная `CJM`
- Roadmap → `task.py` → переменная `ROADMAP`

#### Задание 1.3: Epics + User Stories + DoR

**Что делать:**

```
1. Попросите AI выделить 4–6 Epics и 5–7 User Stories по вашему roadmap
2. Попросите AI составить Definition of Ready (DoR) на 4–6 пунктов
3. Проверьте, что User Stories соответствуют v1.0/v1.1/v2.0
4. Скопируйте результат в task.py
```

**Пример промпта:**

```
По нашему roadmap выдели 4–6 Epics и 5–7 User Stories (формат: "Как [роль], я хочу [действие], чтобы [выгода]").
Также сделай Definition of Ready (DoR) чек-лист (4–6 пунктов) для задач команды.
```

**Куда вставить результат:**

- Epics → `EPICS`
- User Stories → `USER_STORIES`
- DoR → `DEFINITION_OF_READY`

### Этап 2: ADR + HLD

#### Задание 2.1: Компоненты системы

**Что делать:**

```
1. Обсудите в группе: из чего состоит ваш сервис?
2. Составьте список компонентов
3. Запишите в task.py
```

**Подсказка - основные компоненты:**

- FastAPI Backend (REST API)
- Weather API (OpenWeatherMap)
- PostgreSQL Database (хранение подписок)
- Client Apps (веб/мобильные приложения)

#### Задание 2.2: ADR + Mermaid схема

**Что делать:**

```
1. Попросите AI написать короткий ADR (контекст/решение/альтернативы/последствия)
2. Попросите AI сгенерировать Mermaid диаграмму компонентов
3. Вставьте Mermaid код в онлайн-редактор и убедитесь, что схема рендерится
4. (Опционально) скачайте PNG и положите в репозиторий
5. Скопируйте ADR и Mermaid код в task.py
```

**Пример промпта:**

```
Контекст: WeatherService — REST API. Компоненты: Client (Web/Mobile), Backend (FastAPI), DB (PostgreSQL), External Weather API (OpenWeatherMap).
Ограничения: v1.0 должен быть реализуем за 1–2 дня одним инженером; внешние API могут быть недоступны.
Задача:
1) Напиши ADR (кратко): Контекст, Решение, Альтернативы, Последствия/Trade-offs.
2) Сгенерируй Mermaid диаграмму компонентов (Client → Backend → DB/Weather API), отметь основные эндпоинты v1.0.
Формат: сначала ADR, затем Mermaid код в блоке ```mermaid.
```

**Mermaid онлайн редактор:**

1. Идите на <https://mermaid.live/>
2. Вставьте код в левое поле
3. Справа появится диаграмма
4. Нажмите "Actions" → "Download PNG" для скачивания

**Куда вставить результат:**

- Компоненты → `task.py` → `ARCHITECTURE_COMPONENTS`
- ADR → `task.py` → `ADR`
- Mermaid код → `task.py` → `MERMAID_CODE`
- Ссылку/путь на изображение → `task.py` → `MERMAID_IMAGE_URL`
- Саму картинку → на доску

## 💡 Советы и лайфхаки

### Работа с LLM

**✅ Хорошие практики:**

- Говорите с LLM естественно, как с коллегой
- Уточняйте, если результат неполный
- Просите примеры и объяснения
- Сохраняйте все промпты для дальнейшего анализа

**❌ Чего избегать:**

- Не принимайте первый ответ безусловно
- Не копируйте код вслепую (понимайте, что делаете)
- Не забывайте обсуждать результаты в группе

### Эффективные промпты

**Структура хорошего промпта:**

1. Контекст: что мы делаем
2. Задача: что нужно получить
3. Формат: в каком виде
4. Детали: важные уточнения

**Пример:**

```
Контекст: Мы разрабатываем REST API сервис для погоды
Задача: Создай список User Stories
Формат: В виде "Как [роль], я хочу [действие], чтобы [цель]"
Детали: Сосредоточься на основных фичах: подписка, уведомления, управление
```

### Работа в группе

**Распределение задач:**

- Один участник ведет диалог с LLM
- Другой анализирует результаты
- Третий заполняет task.py
- Четвертый следит за временем

**Рекомендация:**
Меняйтесь задачами на каждом этапе для равномерного участия!

## ⚠️ Частые проблемы и решения

### Проблема: LLM дал слишком общий ответ

**Решение:**

```
Уточните запрос. Добавьте детали:
"Сделай это более конкретно для Telegram-бота"
"Добавь примеры для нашего случая с погодой"
```

### Проблема: Mermaid код не работает

**Решение:**

```
1. Проверьте, что скопировали весь код
2. Попросите LLM: "Исправь синтаксис Mermaid кода"
3. Используйте онлайн-редактор Mermaid для проверки
```

### Проблема: Не хватает времени

**Решение:**

```
1. Работайте параллельно (один спрашивает, другой документирует)
2. Используйте готовые промпты из README
3. Не стремитесь к идеалу - лучше закончить все этапы
```

### Проблема: LLM не понимает запрос

**Решение:**

```
1. Упростите формулировку
2. Разбейте на более мелкие вопросы
3. Дайте пример того, что хотите получить
```
