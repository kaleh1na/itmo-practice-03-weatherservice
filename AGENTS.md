# AGENTS.md для образовательного проекта ИТМО

Этот файл содержит упрощенные инструкции для AI-агентов по работе с образовательным проектом курса "AI-инструменты в жизни инженера: от идеи до продакшена" ИТМО.

## Структура проекта

```
ITMO/
├── practices/              # Практики для студентов
│   ├── practice_01/        # Первая практика
│   │   ├── README.md       # Описание практики для студентов
│   │   ├── STUDENT_GUIDE.md # Подробное руководство для студентов
│   │   └── task.py         # Шаблон для выполнения задания
│   └── ...                 # Другие практики
├── solutions/              # Эталонные решения для преподавателей
│   ├── practice_01/        # Решение для первой практики
│   │   ├── solution.py     # Полное эталонное решение
│   │   └── README_INSTRUCTORS.md # Руководство для преподавателей
│   └── ...                 # Другие решения
├── specs/                  # Спецификации проекта
│   ├── spec.md             # Основная спецификация проекта
│   ├── plan.md             # План реализации
│   ├── tasks.md            # Задачи проекта
└── AGENTS.md               # Этот файл
```

## Формат образовательных материалов

### Для студентов (practices/)

Каждая практика должна содержать:

1. **README.md** - Описание практики:
   - Связь с лекцией
   - Цель практики
   - Проект и его описание
   - Итоговые артефакты
   - Структура практики с таймингом
   - Домашнее задание
   - Критерии оценки

2. **STUDENT_GUIDE.md** - Подробное руководство:
   - Пошаговые инструкции
   - Примеры промптов
   - Решение типичных проблем
   - Чек-лист для сдачи

3. **task.py** - Шаблон для выполнения:
   - Структурированные секции для заполнения
   - Места для промптов и результатов
   - Функция экспорта артефактов

### Для преподавателей (solutions/)

Каждое решение должно содержать:

1. **solution.py** - Полное эталонное решение:
   - Заполненный шаблон task.py
   - Референсные артефакты
   - Код (если применимо)

2. **README_INSTRUCTORS.md** - Руководство для преподавателей:
   - Структура решения
   - Критерии оценки
   - Рекомендации по проверке
   - Типичные ошибки студентов
   - Шкала оценивания

## Языковые стандарты

- Все образовательные материалы на русском языке
- Технические термины и код остаются на английском
- Комментарии в коде на русском языке

## Принципы создания образовательных материалов

1. **Практическая направленность**:
   - Все задания должны быть выполнимы студентами
   - Артефакты должны быть конкретными и проверяемыми
   - Упор на результат, а не на процесс

2. **Ясность и структурированность**:
   - Четкие инструкции с примерами
   - Логическая последовательность заданий
   - Визуальные разделители между секциями

3. **Образовательная ценность**:
   - Каждое задание должно обучать чему-то конкретному
   - Наличие рефлексии и ретроспективы
   - Связь с предыдущими и будущими практиками

## Иерархия правил и Конституция

### 1. Конституция (`.specify/memory/constitution.md`) — ЗАКОН
**Верховный документ проекта.** Определяет фундаментальные принципы ("ЧТО" и "ПОЧЕМУ").
- Содержит незыблемые правила (TDD, Library-First, Priority of Russian Language).
- **Агент ОБЯЗАН** прочитать этот файл перед началом работы.
- Любое решение, противоречащее Конституции, считается ошибочным.

### 2. AGENTS.md (Этот файл) — ИНСТРУКЦИЯ
**Операционное руководство.** Определяет способы реализации ("КАК" и "ГДЕ").
- Содержит шаблоны, структуру папок, стандарты оформления.
- Подчиняется Конституции.

> **Важно:** Полный список актуальных принципов находится в файле `.specify/memory/constitution.md`. Руководствуйтесь им как первоисточником.
